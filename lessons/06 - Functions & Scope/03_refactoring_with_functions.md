# Refactoring Earlier Scripts Into Functions

> **Source:** New content
>
> **Notes:** This is a practical exercise section, not primarily a concepts section. Take a script students wrote in an earlier week (e.g., the decision-making script from Week 3 or the loop-based script from Week 4) and walk through refactoring it step by step into named functions. Goals: students should understand (1) how to identify what should become a function (a logical chunk that does one thing), (2) how to name a function well, (3) what top-level "driver" code looks like vs. function definitions. Brief intro to docstrings (one-line `"""Description."""` style). Optionally introduce an `if __name__ == "__main__":` guard as a professional pattern.
