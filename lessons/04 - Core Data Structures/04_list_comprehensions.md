# List Comprehensions

> **Source:** Ported from Python Essentials — PE 3.3 (Python List Comprehensions)
>
> **Porting notes:** Take PE 3.3 list comprehensions content. Include: basic syntax `[expr for x in iterable]`, filtering with `if`, side-by-side comparison with an equivalent `for` loop, practical examples. **Leave out:** generator expressions (the `(x for x in ...)` syntax from PE 3.3) — they add complexity without immediate payoff at this stage and can be introduced in Python Essentials. Emphasize readability: a comprehension is better when it's shorter AND clearer; use a regular loop when the logic is complex.
