# When to Write Your Own Code vs. Import a Module

> **Source:** New content — not in Python Essentials
>
> **Notes:** This is a judgment and professional-skills section. Cover: the three tiers (standard library → well-known third-party package → write it yourself) and the decision logic for each; how to search for and evaluate a package (stars, activity, docs quality, license); a brief look at PyPI; a short example of a task where importing is clearly better (e.g., parsing dates), one where writing your own makes sense (e.g., a very small utility), and one that's in between. The goal is to give students the mental model they'll need when they hit their first "should I just pip install something?" moment.
