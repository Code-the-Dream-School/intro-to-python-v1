## Phase 1: Core Requirements

> All students complete Phase 1.

**What to build:** A Python program that fetches data from a public web API, transforms it into structured Python data, and handles errors robustly.

**Requirements:**
- **External data integration:** Connect to a public web API using the `requests` library
- **Data transformation:** Parse the JSON response into a list of dictionaries
- **Defensive programming:** Implement `try/except` blocks to handle connection errors and runtime errors
- **Modular design:** Refactor all logic into reusable functions with clear parameters and return values

---

## Phase 2: CLI Tool (required)

> All students complete Phase 2 after completing Phase 1.

**What to build:** A command-line interface that lets a user query or interact with the API data in real time.

**Requirements:**
- Accept user input (via `input()` prompts or `sys.argv` / `argparse` arguments)
- Display results in a readable, formatted way
- Include at least one meaningful interaction: e.g., filter by a field, look up a specific item, or compare multiple results
